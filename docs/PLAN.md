## Спринт 1 — «Препресс-точность» (P0)

Цель: оператор видит пригодность нанесения к печати **до** выгрузки и получает корректный под метод production-файл.

### S1.1 — Профили методов печати (#1)
- **Модель** (`types/index.ts`): тип `PrintMethod = "dtf" | "screenprint" | "embroidery"`; поле метода на `Placement` (и опц. дефолт на `PrintArea`). Профиль метода — справочник:
  - **DTF:** `color_mode: "cmyk"`, Pantone — нет, `min_dpi: 300`, вывод — растр/вектор.
  - **Шелкография:** `color_mode: "spot"`, Pantone — да, `min_dpi: 150` (линарт мягче), вывод — вектор + spot-метки.
  - **Вышивка:** контроль не по DPI, а по **минимальной прошиваемой детали** (`min_line_mm ≈ 1`, `min_text_mm ≈ 4`), вывод — спецификация позиции (диджитайз вне инструмента).
- **Валидация** (`lib/catalog/schema.ts`): zod-схема профиля/поля.
- **UI** (`components/editor/SidePanel.tsx`): селектор метода на нанесении.
- **Приёмка:** метод выбирается; профиль подтягивает свои пороги/режим цвета.

### S1.2 — Live-индикатор качества (#2)
- **Логика** (`lib/catalog/dpi.ts` — есть): для DTF/шелкографии бейдж DPI = px макета / реальный размер печати; для вышивки — проверка `min_line_mm`/`min_text_mm`.
- **UI:** бейдж Good / Средне / Низко в `SidePanel`, порог из профиля метода; **не блокирует**.
- **Приёмка:** при ресайзе нанесения бейдж пересчитывается; для вышивки показывает предупреждение о мелких деталях.

### S1.3 — Method-aware production PDF (#3)
- **Экспорт** (`lib/export/buildSceneSvg.ts`, `exportPdf.ts`): production-слой = чистый вектор нанесения без выносок; для шелкографии — spot/Pantone-метки; для DTF — чистый CMYK/RGB; для вышивки — лист позиции/размера.
- **Приёмка:** PDF несёт корректный под выбранный метод production-вывод; обвязка/выноски — в отдельном слое.

### S1.4 — Pre-export чеклист (#32)
- **Новый** `lib/export/preflight.ts` + модалка перед PDF: low-DPI (по методу), нет прозрачного фона, выход за зону, нанесение без метода.
- **Приёмка:** перед экспортом — список проблем с переходом к проблеме; экспорт по подтверждению (не жёсткий блок).

**Выход спринта 1:** оператор за тот же поток получает PDF, готовый под DTF/шелкографию/вышивку, с предупреждениями о качестве до выгрузки.

## Спринт 2 — «Обвязка как объект» (P0)

Цель: размерная обвязка вычисляется автоматически и корректно пересчитывается по ростовке; нанесение привязано к якорям изделия по индустриальным стандартам.

### S2.1 — Обвязка как параметрический объект (#9)
- **Геометрия** (`lib/geometry/dimensionLines.ts`): размерная линия = ссылка на 2 точки (from/to), число вычисляется из геометрии в мм, пересчёт на каждое изменение (не «нарисованные» линии). PDF (`buildSceneSvg`) рендерит из этих же линий.
- **Приёмка:** двигаешь/масштабируешь макет — числа обвязки меняются сами; экспорт берёт те же значения.

### S2.2 — Anchor-пресеты размещения (#11)
- **UI** (`SidePanel` пресеты) + дефолты в мм от якорей изделия (`STANDARD_PLACEMENT` в `view.ts`): центр груди — **верх принта ≈ 75 мм ниже горловины** (как в индустриальных гайдах: от горловины до верха печати), left chest ≈ **89 мм от центра**, ≈ **60 мм ниже горловины**.
- **Приёмка:** кнопки-пресеты ставят нанесение по стандарту относительно якоря (горловина/центр), не относительно холста.

### S2.3 — Grade-rule ΔX/ΔY (#10)
- **Модель/геометрия** (`lib/geometry/gradeRule.ts`, `GradeRule` в `types/index.ts`): правило ростовки как дельты ΔX/ΔY якорей на шаг размера; знак шага = «вверх/вниз от базового»; разворот в `size_anchors` при загрузке каталога; пересчёт якорей и привязанных нанесений по всей сетке через существующий регрейдинг. (`SizeGrade` — устаревшая заглушка, заменена на `GradeRule`.)
- **Тесты** (`vitest`): эталонные кейсы; отступ от горловины — константа на всех размерах.
- **Приёмка:** смена размера двигает якоря и нанесения по правилу; тесты зелёные.

**Выход спринта 2:** обвязка живая и метрически верна на всех ростовках; размещение — по индустриальным якорным стандартам.
