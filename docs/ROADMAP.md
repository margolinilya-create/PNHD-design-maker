## Трек «лекала + точность» (текущий приоритет)

### P0 — калибровка масштаба ✅
- `scale_mm_per_unit` задействован сквозняком: на холсте (flatMm = naturalWidth × scale)
  и в экспорте PDF (флэт обёрнут в `<g transform="scale(s)">`, страница в мм).
- Калибровка в «Линейке»: измеряешь известный элемент → вводишь реальную мм →
  получаешь `scale_mm_per_unit` для записи в лекало (skus.json).
- Тест: PDF масштабирует флэт и держит страницу в мм.

### P1 — точность данных
- **Per-size зоны печати** (`print_areas` градуируются по ростовкам, а не только якоря).
  Модельная доработка: зоны как функция размера.
- **Ручная калибровка растрового макета** (ввод реальной ширины мм / DPI вместо дефолта 120 мм).
- **Live-индикатор DPI** (решено: бейдж Good/Средне/Низко = пиксели макета / реальный размер
  печати; НЕ блокирует) — `SidePanel`/оверлей.

### Заведение первого реального SKU ✅ (FreeFit)
- Пайплайн DXF/AI/PDF → SVG в мм → калибровка → якоря → зоны → ростовки.
- Эталон для регрессий точности.
- Сделано: SKU `tshirt-freefit` из `PNHDTSh_FU_FreeFit.dxf` (AAMA-выкройка R12).
  Скрипты `scripts/dxf_analyze.py` (разбор) и `scripts/dxf_build_sku.py` (сборка):
  деталь ориентируется по долевой нити, нормализуется в мм, якоря и per-size
  ростовки (2XS…4XL) выводятся из геометрии. Зоны печати пока **провизорные**
  (уточняются в редакторе). Перёд/спина/рукав, реальный грейдинг горловины.

### Редактор лекал в приложении (`/admin`) ✅ v1
- Загрузка флэта (SVG/PNG) → разметка на холсте: калибровка масштаба «Линейкой»,
  перетаскиваемые якоря (горловина/ось или низ/центр рукава), зона печати с ресайзом.
- Экспорт: готовый JSON SKU (скачать/копировать) + «черновик» в localStorage —
  модель сразу видна в каталоге для превью.
- `lib/admin/flatDraft.ts` (модель+сборка+валидация), `components/admin/FlatEditorCanvas.tsx`.

### Редактор лекал v2 ✅
- **Мультивид + per-size в одном потоке**: создание (флэт/DXF) → «Продолжить в
  редакторе» открывает собранный SKU в полном `SkuEditor`. Добавление видов
  (перёд/спина/рукав/этикетка), ростовки и per-size геометрии — в одной сессии,
  без save→reopen. Виды без флэта помечены бейджем «нет флэта», холст показывает
  заглушку вместо падения. DXF «Собрать SKU → редактор» открывает все
  виды+ростовки для выверки провизорных зон.

### Импорт лекал — дальше
- Персистентность черновиков на сервер (когда вернёмся к Supabase) + сохранение
  флэта как файла (сейчас флэт в черновике — data URL, для PDF нужен файл в seed).

## Паритет с Printful ✅ (раунд 1)

- **Маскирование макета по зоне + предупреждение** ✅: макет обрезается по своей печатной
  зоне (клип-группа на холсте, `clipPath` в PDF); при выходе зона подсвечивается красным.
- **Пресеты позиции** ✅: «Центр X / Центр зоны / Вверх / Вниз» в инспекторе (по оси изделия
  и safe-inset зоны).
- **Мультизона на виде** ✅: зона резолвится по `print_area_id` нанесения
  (`findPrintArea`/`viewZone(view,size,areaId)`); селектор зоны при загрузке.
  Оба рукава (`sleeve_left`/`sleeve_right`) и **этикетка как доп. зона** — на FreeFit
  (спина: зоны «Спина» + «Этикетка»).

### Осталось по теме
- Этикетки шеи как отдельные **виды** (а не доп. зоны) — если нужен отдельный флэт этикетки.
- Зоны печати у FreeFit — **выверить под реальную печать** (сейчас провизорные).

## Сохранение проектов и общий каталог ✅ (Supabase-ready)
- `lib/persistence/projects.ts`: проекты save/load/list/delete; стор `snapshot()/restore()`;
  UI в панели «Проект».
- `lib/persistence/models.ts`: пользовательские модели каталога (из DXF/редактора)
  save/list/delete — те же бэкенды. SkuPicker грузит их и даёт удалять; бейдж «моя».
- Бэкенд: **Supabase** (если `NEXT_PUBLIC_SUPABASE_URL/ANON_KEY`), иначе **localStorage**.
- Активация облака: выбрать Supabase-проект → миграции (таблицы `pinhead_projects`,
  `pinhead_models`) + RLS → env в Vercel. Тогда проекты и каталог — общие для всех.
